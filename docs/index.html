MOVIE-API Documentation

Welcome to the Movie-Api docs.

Our movie API is set to build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies. It is necessary to login (and therefore, register) before doing absolutely anything. To register, go to /register and add name, password, email, and a birth date (format yyyy/mm/dd).

Getting Started

The first thing to get started is to connect to the server from your application: https://myflix-lounge.herokuapp.com/

If you are using React, you can use Axios. If you use Angular, refer to its httpRequest System.

URL Endpoints

  • GET "/API/movies"

    Deploys as a response a list of all the movies in our Database. Requires registration and login (see POST login and POST register).

  • GET "/API/movies/:title"

    Deploys as a response the selected movie by title. Requires registration and login (see POST login and POST register).

  • POST "/users"

    Allows to sign up. Requires a name, password, email, and a birth date (format yyyy/mm/dd). Responses with the brand-new created user in JSON.

  • POST "/users/:Username/Movies/:MovieID"

    Allows you to add a movie to your favourite movies list. Requires registration and login (see POST login and POST register)

  • POST "/login"

    Allows you to access to your account by inserting user name and password. Afterwards, you will receive a JWT with which you will be allowed to enter to the rest of the endpoints.

  • PUT "/users/:Name"

    Updates user's info. Expects JSON in this format: { Name: String, (required) Password: String, (required) Email: String, (required) Birthday: Date }. Requires registration and login (see POST login and POST register)

  • DELETE "/users/:Username/Movies/:MovieID"

    Allows you to remove a movie to your favourite movies list. Requires registration and login (see POST login and POST register)

  • DELETE "/users/:Username"

    Allows you to remove your user. Requires registration and login (see POST login and POST register)
