Movie API documentation

API goal

Our movie API is set to build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

URL Endpoints

  • GET "/API/movies": Deploys as a response a list of all the movies in our Database
  • GET "/API/movies/:title" : Deploys as a response the selected movie by title
  • POST "/register" : allows to sign up. Requires a name, password, email, and a birth date (format yyyy/mm/dd). Responses with the brand-new created user in JSON.
  • POST "/users/:Username/Movies/:MovieID" : allows you to add a movie to your favourite movies list
  • POST "/login" : allows you to access to your account by inserting user name and password.
  • PUT "/users/:Name" : updates user's info. Expects JSON in this format: { Name: String, (required) Password: String, (required) Email: String, (required) Birthday: Date }
  • DELETE "/users/:Username/Movies/:MovieID" : allows you to remove a movie to your favourite movies list
  • DELETE "/users/:Username" : allows you to remove your user.
