Movie API documentation

API goal

Our movie API is set to build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

URL Endpoints

  • GET "/API/movies": Deploys as a response a list of all the movies in our Database
  • GET "/API/movies/:title" : Deploys as a response the selected movie by title
  • GET "/users/:name/:id" : responses access to your user profile in JSON.
  • POST "/register" : allows to sign up. Requires a name and a password. Responses with the brand-new created user in JSON
  • PUT "/users/:name/:id/:newName" : allows to change the name. Responses with the user in JSON, with the new name
